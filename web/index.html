Cloudera ECS AI Ops Chatbot

An air-gapped Kubernetes operations assistant for Cloudera ECS, powered by a local LLM (Qwen3-8B), LangGraph agentic loop, and ChromaDB RAG.

💡 Quick start: Type a question in the ECS AI Ops tab — e.g. "is my cluster having any issues?" — and press Enter.
📖 Prerequisites: Python 3.12 is required.

How it works

Each query runs through an agentic ReAct loop driven by the LLM — not by hardcoded rules. The LLM reads a Tool Selection Guide in its system prompt, picks the right tools, observes the results, and decides whether to call more tools or synthesise a final answer.

⚠ The assistant is read-only by default. It will never restart, delete, or modify cluster resources.

Example queries

  • Which node has GPUs available and in use?
  • Which secret in cdp has user credentials?
  • List all PVCs not bound across all namespaces
  • Show warning events in cdp namespace

Configuring the Cluster

Click the ⚙ gear icon → Cluster tab. Paste your kubeconfig YAML and click Apply. The cluster host appears in the header on success. No restart needed — you can switch clusters at any time.

curl -s -X POST http://localhost:9000/api/kubeconfig \
     -H 'Content-Type: application/json' \
     -d "{\"kubeconfig\": \"$(cat ~/.kube/config | python3 -c 'import sys,json; print(json.dumps(sys.stdin.read()))')\"}"

Knowledge Base (RAG)

The assistant cross-references live cluster data with your runbooks, known-issues docs, and SOPs via ChromaDB RAG.

Via the Settings UI

  1. Click ⚙ → Knowledge Base tab.
  2. Drop .md, .pdf, or .txt files onto the drop zone.
  3. Optionally tick Force re-ingest to overwrite existing entries.
  4. Click Ingest.

Via server-side directory

python3 app.py --ingest ./docs
python3 app.py --ingest ./docs --force   # re-ingest all

Customising the System Prompt

The LLM's tool selection, multi-hop behaviour, and persona are all controlled by config/system_prompt.txt. Key sections:

  • Tool Selection Guide — which tools to call for each query type
  • Multi-hop Reasoning — when to chain calls (e.g. OOMKilled → check quotas)
  • CRITICAL: Never Narrate — prevents the LLM from suggesting tools instead of calling them

Hot-reload after editing — no restart needed:

curl -s -X POST http://localhost:9000/api/reload-prompt

Security Notes

  • All typed K8s tools are read-only by design.
  • kubectl_exec is read-only by default. Set KUBECTL_ALLOW_WRITES=true to enable write ops.
  • Secret values are hidden by default — toggle in ⚙ Settings → Security. Preference persists per browser.
  • Never expose this service publicly — it has direct cluster read access.
  • Restrict the env file: chmod 600 env
